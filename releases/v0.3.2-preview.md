# v0.3.2-preview

## tl;dr

- Packages MCP streamable HTTP as the preview stack sidecar path.
- Existing stdio MCP configs still work.
- Server image: `kexinlu/mesh-context-server:0.3.2-preview`.
- MCP sidecar image: `kexinlu/mesh-memory:mcp-0.3.2-preview`.

## MCP Sidecar

After minting a token, set it in `.env`:

```bash
MESH_API_TOKEN=mesh_xxx
```

Start the sidecar profile:

```bash
COMPOSE_PROFILES=mcp-http docker compose up -d
```

Connect MCP streamable HTTP clients to:

```text
http://127.0.0.1:9081/mcp
```

Multiple clients can connect concurrently. The sidecar uses in-memory MCP
sessions; each initialized client receives a distinct `mcp-session-id`.
Restarting the sidecar clears those sessions, so clients should reconnect.

The sidecar forwards tool calls to `mesh-context-server` GraphQL using the
configured `MESH_API_TOKEN`. For per-client tenant/role isolation, run separate
sidecar instances with different tokens or keep using stdio bridge containers
per client.
