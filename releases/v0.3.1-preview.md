# v0.3.1-preview

## tl;dr

- Adds a streamable HTTP MCP endpoint in the MCP image.
- Existing stdio MCP configs keep working.
- Server image: `kexinlu/mesh-context-server:0.3.1-preview`.
- MCP image: `kexinlu/mesh-memory:mcp-0.3.1-preview`.

## MCP HTTP Endpoint

The MCP image now supports two transports:

- default stdio transport for local MCP clients.
- streamable HTTP transport at `/mcp` when `MESH_MCP_TRANSPORT=http`.

Run directly:

```bash
docker run --rm --network host \
  -e MESH_API_URL=http://127.0.0.1:9080/graphql \
  -e MESH_API_TOKEN="$MESH_API_TOKEN" \
  -e MESH_MCP_TRANSPORT=http \
  -e MESH_MCP_BIND=127.0.0.1:9081 \
  kexinlu/mesh-memory:mcp-0.3.1-preview
```

Endpoint:

```text
http://127.0.0.1:9081/mcp
```

Or use the optional Compose profile after adding `MESH_API_TOKEN` to `.env`:

```bash
COMPOSE_PROFILES=mcp-http docker compose up -d
```

## Upgrade

Pull the new images/config:

```bash
docker compose pull
docker compose up -d
```

Only enable the `mcp-http` profile if your client supports MCP streamable HTTP
and you have set `MESH_API_TOKEN`.
